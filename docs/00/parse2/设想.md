### Q1: Plugin 怎么实现？

两种方式：

1. Native Plugin：直接实现接口
2. MCP Plugin：通过 MCP 协议接入现有 Agent

*// Plugin 接口（统一）*

```
interface Plugin {
  readonly id: string
  readonly name: string
  call(input: unknown): Promise<unknown>
  getSchema(): { input: Schema, output: Schema }
}
```

### Q2: WorkflowNode 的作用？

核心作用：统一 I/O 格式

​	Plugin A → I/O 格式: { x: number, y: number }

​	Plugin B → I/O 格式: { a: string, b: string }

​	WorkflowNode A → 标准 I/O: { left: number, right: number }

​	WorkflowNode B → 标准 I/O: { data: string }

数据流映射：

​	  WorkflowNode A.right → WorkflowNode B.data

  	用户输入.left → WorkflowNode A.left
