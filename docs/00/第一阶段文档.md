## 一、项目定位与愿景

### 1.1 核心定位

一个**本地运行的 AI Agent 桌面应用**，让任何人都可以：

- 不联网本地使用已有功能
- 通过 MCP 接口获取/分享功能插件
- 享受去中心化的 Agent 服务

### 1.2 长远愿景（去中心化 Agent 网络）

```
用户拥有 Agent 个体源
       ↓
从 Agent 区块链网获取功能插件（如健身插件）
       ↓
使用公共功能时，有非中心化共识监管机制
       ↓
所有功能开源，避免商业化，提升大众生活品质
```

### 1.3 当前 MVP 定位

**简化为"本地 AI 助手桌面应用"**：

- 基于多模型支持（OpenAI/Claude/Qwen 等）
- 本地对话 + 工具调用
- 对话历史管理与压缩

## 四、核心设计亮点

### 4.1 六边形架构（Ports & Adapters）

将业务逻辑与外部依赖解耦：

```
┌─────────────────────────────────────────────────────────┐
│                      应用核心 (Domain)                    │
│   ┌─────────────────────────────────────────────────┐   │
│   │   业务逻辑 / 用例 (AgentService)                │   │
│   └─────────────────────────────────────────────────┘   │
│                          │                              │
│   端口 (Port) ──────────┼────────── 端口 (Port)       │
│   输入端口              │              输出端口         │
└─────────────────────────┼───────────────────────────────┘
                          │
         ┌────────────────┼────────────────┐
         ▼                ▼                ▼
   ┌──────────┐    ┌──────────┐    ┌──────────┐
   │  UI 适配器│    │ LLM 适配器│    │存储适配器 │
   └──────────┘    └──────────┘    └──────────┘
```

### 4.2 模型网关设计（适配器模式 + 工厂模式）

```typescript
// LLMManager 充当工厂，根据 provider 创建对应适配器
class LLMManager {
  createAdapter(key: ApiKey): LLMPort {
    switch (key.provider) {
      case 'openai': return new OpenAIAdapter(...);
      case 'anthropic': return new ClaudeAdapter(...);
      case 'qwen': return new QwenAdapter(...);
      // 可扩展更多厂商
    }
  }
}
```

### 4.3 对话压缩机制

```typescript
// 每 20 轮自动压缩，防止上下文过长
if (messageCount % 20 === 0) {
  await compressionService.compressConversation(id);
}

// 压缩服务负责：
// 1. 获取历史消息
// 2. 调用 LLM 生成摘要
// 3. 标记旧消息为已压缩
// 4. 更新对话摘要
```

### 4.4 状态同步架构

```
electron-store (持久化)
       ↓ IPC 调用
Pinia (前端状态管理)
       ↓ 计算属性
UI 组件 (响应式更新)
```

**关键修复**：确保 `loadLLMConfig()` 同步 `activeKeyId` 和 `activePromptId`，避免切换页面后状态丢失。

## 五、关键问题与解决方案

### 5.1 模型/Prompt 选择器不同步

**问题**：切换页面后，模型和 Prompt 下拉框显示"未选择"或显示第一个选项。

**原因**：`loadLLMConfig()` 只加载 `llmConfig`，没有同步 `activeKeyId`。

**解决方案**：
```typescript
async function loadLLMConfig() {
  const [providerList, config, activePrompt] = await Promise.all([
    configApi.listProviders(),
    configApi.getActiveLLMConfig(),
    configApi.getActivePrompt(),
  ]);
  
  // 同步 activeKeyId
  if (config?.keyId) {
    activeKeyId.value = config.keyId;
  }
  
  // 同步 activePromptId
  if (activePrompt?.id) {
    activePromptId.value = activePrompt.id;
  }
}
```

### 5.2 历史对话时间显示

**需求**：只显示开始时间，标题不斜体改为黑色。

**解决方案**：
- 修改 `formatTimeRange()` 只返回开始时间
- 移除 `.item.ended .name` 的 `font-style: italic`
- 添加 `color: #000` 使标题始终为黑色
