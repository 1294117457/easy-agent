✅ 前端 UI + Agent Pet           ⏳ 条件分支并行

                                 ⏳ 模板市场

## Phase 1 — MVP 基础（目标：一个能跑的产品）

### 1.1 阶段目标

实现最小可用产品：用户配置 API Key、接入 MCP 插件、编排工作流、对话交互。

### 1.2 功能模块

#### 模块 1.1：项目骨架

| 功能 | 描述 | 交付物 |
|------|------|--------|
| Monorepo 结构 | packages/server + packages/frontend | 目录结构 |
| 前后端工程初始化 | Vite + Vue 3 TS / Express + TS | 可运行项目 |
| 数据库初始化 | SQLite 建表 | 数据库文件 |
| 前端基础 UI | 布局 + 导航 + 主题 | 可视化界面 |

#### 模块 1.2：模型适配器

| 功能 | 描述 | 交付物 |
|------|------|--------|
| 抽象适配器接口 | `LLMAdapter` 接口定义 | `adapters/base.ts` |
| OpenAI 适配器 | `OpenAIAdapter` 实现 | `adapters/openai-adapter.ts` |
| Anthropic 适配器 | `AnthropicAdapter` 实现 | `adapters/anthropic-adapter.ts` |
| 适配器工厂 | 根据 API Key 配置动态创建 | `adapters/index.ts` |
| API Key 配置 | 添加/删除/列表（加密存储） | 设置页面 |

#### 模块 1.3：MCP 插件接入

| 功能 | 描述 | 交付物 |
|------|------|--------|
| MCP Server 添加 | STDIO + SSE 两种模式 | 插件管理页 |
| 工具发现 | 接入后自动列出工具 | 工具列表 |
| 启用/禁用 | 开关控制工具可用性 | UI 开关 |
| MCP 客户端 | STDIO/SSE 连接管理 | `mcp-client.ts` |

#### 模块 1.4：LangGraph 核心

| 功能 | 描述 | 交付物 |
|------|------|--------|
| Agent State 定义 | 状态结构定义 | `types/agent-state.ts` |
| LLM 路由节点 | 模型决策是否调用工具 | `nodes/llm-router.ts` |
| 工具执行节点 | MCP 工具调用 | `nodes/tool-executor.ts` |
| 图编译 | 串联节点 → 可执行图 | `graph/compiler.ts` |
| Agent 服务 | 封装 LangGraph 调用 | `services/agent.ts` |

#### 模块 1.5：工作流编排

| 功能 | 描述 | 交付物 |
|------|------|--------|
| 节点拖拽 | 从工具面板拖到画布 | Vue Flow |
| 节点配置 | 点击节点配置参数 | 配置面板 |
| 连线编排 | 拖拽连接节点 | Vue Flow Edge |
| 保存/加载 | 工作流 CRUD | 编排页面 |
| 执行 | 触发 LangGraph 执行 | 执行按钮 |

#### 模块 1.6：SSE 流式交互

| 功能 | 描述 | 交付物 |
|------|------|--------|
| 后端 SSE 推送 | token / done / error 事件 | `sse-emitter.ts` |
| 前端 SSE 接收 | fetch + ReadableStream 解析 | `chat.ts` |
| Agent Pet 联动 | 根据事件切换心态 | `AgentPet.vue` |

### 1.3 验收标准

- [ ] 用户可以配置 OpenAI API Key 并正常对话
- [ ] 用户可以接入一个 MCP Server 并调用其工具
- [ ] 用户可以编排一个简单工作流并执行
- [ ] Agent Pet 实时反馈执行状态
- [ ] 页面基本可操作，无明显阻塞

## Phase 2 — 功能完善（目标：一个好用的产品）

### 2.1 阶段目标

Phase 1 的基础上增加用户真正需要的功能：对话历史、Prompt 模板、执行日志。

### 2.2 功能模块

#### 模块 2.1：对话历史

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 对话列表 | 查看所有历史对话 | P0 |
| 重新加载 | 点击历史对话继续 | P0 |
| 删除对话 | 删除不需要的对话 | P0 |
| 重命名 | 修改对话名称 | P1 |
| 按时间筛选 | 快速定位历史 | P2 |

#### 模块 2.2：Prompt 模板管理

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 创建模板 | 输入名称 + System Prompt | P0 |
| 模板列表 | 查看已创建的模板 | P0 |
| 选择模板 | 对话时选择使用哪个模板 | P0 |
| 删除/编辑 | 维护已有模板 | P1 |
| 预设模板 | 内置几个常用模板 | P1 |

#### 模块 2.3：执行日志

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 步骤记录 | 记录每步执行了什么 | P0 |
| 输入/输出展示 | 每步的入参和结果 | P0 |
| 执行状态 | running / success / failed | P0 |
| 错误详情 | 失败时展示原因 | P1 |

#### 模块 2.4：条件分支

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 条件节点 | if / else 分支 | P0 |
| 条件评估 | LLM 判断条件是否满足 | P1 |
| 分支可视化 | 画布上区分不同分支 | P0 |
| 多分支 | 支持 3 个以上分支 | P2 |

#### 模块 2.5：错误处理与重试

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 工具调用失败 | 捕获错误，通知前端 | P0 |
| 模型调用失败 | 重试机制（最多 2 次） | P1 |
| Agent Pet 联动 | 错误时切换 error 状态 | P0 |
| 错误日志 | 持久化错误信息 | P1 |

### 2.3 验收标准

- [ ] 用户可以查看、加载、删除历史对话
- [ ] 用户可以创建和管理 Prompt 模板
- [ ] 用户可以看到工作流的每步执行日志
- [ ] 工具调用失败有明确的错误提示

## Phase 3 — 体验优化（目标：一个好看好用的产品）

### 3.1 阶段目标

完善用户体验：多模型支持、工作流导入导出、性能优化。

### 3.2 功能模块

#### 模块 3.1：多模型支持

| 功能 | 描述 | 优先级 |
|------|------|--------|

| DeepSeek 适配器 | 接入 DeepSeek | P1 |
| 模型切换 | 对话中动态切换模型 | P0 |
| 模型信息展示 | 显示当前使用的模型 | P0 |

#### 模块 3.2：工作流导入/导出

| 功能 | 描述 | 优先级 |
|------|------|--------|
| JSON 导出 | 工作流导出为 JSON 文件 | P0 |
| JSON 导入 | 加载外部 JSON 工作流 | P0 |
| 复制分享 | 生成可分享的工作流链接 | P1 |
| 工作流重命名/描述 | 完善元信息 | P1 |

#### 模块 3.3：长期记忆（RAG）

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 向量库集成 | ChromaDB / FAISS | P1 |
| 文档上传 | 上传知识库文档 | P1 |
| 记忆检索 | 对话时自动检索相关记忆 | P1 |
| 记忆管理 | 查看/删除已存储的记忆 | P2 |

#### 模块 3.4：定时触发与 Webhook

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 定时执行 | 设置时间自动运行工作流 | P2 |
| Webhook 触发 | 外部 HTTP 请求触发 | P2 |
| Webhook 通知 | 工作流完成后通知外部 | P2 |

#### 模块 3.5：性能优化

| 功能 | 描述 | 优先级 |
|------|------|--------|
| MCP 连接复用 | 避免每次调用重建连接 | P1 |
| 模型响应缓存 | 相同请求缓存结果 | P2 |
| 前端虚拟列表 | 大量消息时不卡顿 | P1 |
| 懒加载路由 | 非首屏组件延迟加载 | P1 |

### 3.3 验收标准

- [ ] 支持至少 4 个模型厂商（OpenAI / Claude / Gemini / DeepSeek）
- [ ] 工作流可以导出为 JSON 并重新导入
- [ ] 页面加载 < 2s，对话响应体感流畅
- [ ] 有基础的 RAG 能力（上传文档 → 对话时自动检索）
