### 问题 1: 已结束对话仍可发送消息

#### 问题描述

当前实现中，用户可以在已结束的对话中发送新消息。这违反了对话的"结束"语义。

#### 当前代码

```typescript
// src/stores/chat.ts - sendMessage 函数
async function sendMessage(content: string) {
  // 验证有选中的对话
  if (!currentConversationId.value) {
    await newConversation();
  }

  // ❌ 缺少：检查当前对话是否已结束
  // 直接发送消息，导致已结束对话也能继续对话
  const conversationId = currentConversationId.value!;
  // ...
}
```

#### 问题分析

```
当前行为：
用户选择已结束的对话 → 加载历史消息 → 发送消息 → 消息被添加到已结束的对话

期望行为：
用户选择已结束的对话 → 显示历史（只读）→ 发送消息 → 自动创建新对话
```

#### 修改方案

```typescript
// src/stores/chat.ts - sendMessage 函数
async function sendMessage(content: string) {
  // 验证有选中的对话
  if (!currentConversationId.value) {
    await newConversation();
    return;
  }

  // ✅ 新增：检查当前对话是否已结束
  const currentConv = conversations.value.find(
    (c) => c.id === currentConversationId.value
  );
  if (currentConv?.endedAt) {
    // 对话已结束，自动创建新对话
    console.log('[ChatStore] 当前对话已结束，自动创建新对话');
    await newConversation();
  }

  // 继续正常的发送逻辑
  // ...
}
```

#### 补充：历史侧边栏 UI 优化

建议在历史对话列表中，区分"进行中"和"已结束"的对话：

```vue
<!-- src/components/chat/ConversationList.vue -->
<div
  v-for="conv in conversations"
  :class="['conversation-item', { ended: conv.endedAt }]"
>
  <span class="name">{{ conv.name }}</span>
  <span v-if="conv.endedAt" class="ended-badge">已结束</span>
</div>
```

### 问题 2: System Prompt 未生效

#### 问题描述

用户配置的 System Prompt 没有被正确应用到对话中，AI 回答时没有遵循 Prompt 中的指令。

#### 问题分析

System Prompt 未生效可能有以下几个原因：

| 原因 | 检查方法 | 解决方式 |
|-----|---------|---------|
| 没有创建 Prompt | 检查数据库 prompts 表 | 用户需要先创建 Prompt |
| Prompt 未激活 | `getActivePrompt()` 返回 null | 自动激活第一个 Prompt |
| Prompt 内容为空 | `systemPrompt` 字段为空字符串 | 使用默认 Prompt |
| Prompt 创建后未同步激活状态 | `is_active` 字段不一致 | `createPrompt` 后自动激活 |

#### 当前代码分析

```typescript
// src/stores/config.ts - createPrompt 函数
async function createPrompt(data) {
  const created = await configApi.createPrompt(data);
  prompts.value.push(created);
  return created;
  // ❌ 问题：创建后没有激活，也没有提示用户
}
```

```typescript
// electron/core/application/AgentService.ts
const activePrompt = this.storagePort.getActivePrompt();
const systemPrompt = activePrompt?.systemPrompt || '';

// 如果没有激活的 Prompt，会使用空字符串
if (!systemPrompt) {
  console.warn('[AgentService] 警告：没有配置 System Prompt');
}
```

#### 修改方案

**方案 A：在前端创建 Prompt 后自动激活第一个**

```typescript
// src/stores/config.ts - createPrompt 函数
async function createPrompt(data) {
  const created = await configApi.createPrompt(data);
  prompts.value.push(created);

  // ✅ 新增：如果没有激活的 Prompt，自动激活这个
  if (!activePromptId.value) {
    await setActivePrompt(created.id);
  }

  return created;
}
```

**方案 B：在后端创建默认 Prompt**

```typescript
// electron/core/adapters/storage/sqlite.adapter.ts - 构造函数

  // ...
  this.ensureDefaultPrompt();
}

private ensureDefaultPrompt(): void {
  const prompts = this.listPrompts();
  if (prompts.length === 0) {
    // 创建默认 Prompt
    this.createPrompt({
      name: '默认',
      description: '系统默认 Prompt',
      systemPrompt: '你是 EasyAgent，一个有用的 AI 助手。',
      isBuiltin: true,
    });
    // 自动激活

    this.setActivePrompt(defaultPrompt.id);
  }
}
```

### 问题 3: 消息重复添加到历史

#### 问题描述

当前实现中，发送给 LLM 的消息列表中包含了重复的内容，导致 AI 可能重复回答。

#### 当前代码

```typescript
// electron/core/application/AgentService.ts
async sendMessage(conversationId, userInput, callbacks) {
  // 获取历史消息
  const history = this.storagePort.getMessages(conversationId);
  // history = [msg1_user, msg1_assistant, msg2_user, msg2_assistant]

  // 排除当前消息（如果存在）
  const historyWithoutCurrent = history.slice(0, -1);
  // 问题：这里假设 history 最后一条是当前用户消息，但实际上可能不是

  // 只取最近 10 条
  const recentHistory = historyWithoutCurrent.slice(-10);

  // 构建消息
  const langchainMessages = recentHistory.map(m =>
    m.role === 'user'
      ? new HumanMessage(m.content)
      : new AIMessage(m.content)
  );

  // ⚠️ 问题：这里的 recentHistory 已经是包含当前消息的历史
  // 如果 history 最后一条是用户消息（当前消息），则 recentHistory 已经包含了它
  // 然后又 push 了 userInput，导致当前消息被添加两次

  langchainMessages.push(new HumanMessage(userInput));
  // 现在 langchainMessages = [...recentHistory(含当前消息), 当前消息]
  // = 重复了！
}
```

#### 问题分析

```
history = [msg1_user, msg1_assistant, msg2_user, msg2_assistant]
                                                        ↑ 最后一条是用户消息

recentHistory = [msg1_user, msg1_assistant, msg2_user, msg2_assistant]
  // 因为最后一条是用户消息，所以 recentHistory = historyWithoutCurrent

langchainMessages.push(new HumanMessage(userInput))
  // 导致 msg2_user 被添加了两次！
```

#### 修改方案

**关键：判断 history 最后一条是否是当前用户消息**

```typescript
// electron/core/application/AgentService.ts
async sendMessage(conversationId, userInput, callbacks) {
  // 获取历史消息
  const history = this.storagePort.getMessages(conversationId);

  // ✅ 新增：判断最后一条消息是否是当前用户消息
  const lastMessage = history[history.length - 1];
  const isLastMessageFromUser = lastMessage?.role === 'user' &&
                                lastMessage?.content === userInput;

  // 如果最后一条是当前用户消息，不排除；否则排除最后一条
  const historyToUse = isLastMessageFromUser ? history : history.slice(0, -1);

  // 只取最近 10 条
  const recentHistory = historyToUse.slice(-10);

  // 构建消息
  const langchainMessages = recentHistory.map(m =>
    m.role === 'user'
      ? new HumanMessage(m.content)
      : new AIMessage(m.content)
  );

  // ✅ 始终添加当前用户消息
  langchainMessages.push(new HumanMessage(userInput));

  // 构建最终消息列表
  const allMessages = [
    ...(systemPrompt ? [new SystemMessage(systemPrompt)] : []),
    ...langchainMessages,
  ];

  // ...
}
```

**更简洁的方案：直接从数据库获取不含当前消息的历史**

```typescript
// electron/core/application/AgentService.ts
async sendMessage(conversationId, userInput, callbacks) {
  // ✅ 直接获取历史，不包含当前消息
  // 前提：前端在调用 sendMessage 之前还没有写入数据库
  const history = this.storagePort.getMessages(conversationId);
  const recentHistory = history.slice(-10);

  // 构建消息
  const langchainMessages = recentHistory.map(m =>
    m.role === 'user'
      ? new HumanMessage(m.content)
      : new AIMessage(m.content)
  );

  // ✅ 始终添加当前用户消息
  langchainMessages.push(new HumanMessage(userInput));

  // ... 后续逻辑
}
```

### 问题 4: 前后端状态不同步

#### 问题描述

前端 `messages.value` 和后端 `storagePort.getMessages()` 可能不一致，导致显示和实际发送的内容不匹配。

#### 问题分析

```
时序问题：

1. 用户发送消息
2. 前端立即显示：messages.value.push(userMessage)  → 用户看到消息
3. 调用 chatApi.send()
4. 后端发送消息给 LLM
5. LLM 返回结果
6. 后端保存：storagePort.appendMessage(assistant)
7. 前端收到：onToken 回调 → 显示助手消息
8. 前端收到：onDone 回调

问题：如果步骤 3-7 之间，用户又发送了另一条消息，
可能导致消息顺序混乱或消息被错误地添加到历史中。
```

#### 修改方案

**方案 A：前端不维护独立的消息状态，从后端获取**

```typescript
// src/stores/chat.ts - sendMessage 函数
async function sendMessage(content: string) {
  // ... 验证逻辑

  // ✅ 不在本地添加用户消息，等待后端返回
  isLoading.value = true;

  try {
    await chatApi.send(conversationId, content);
    // ✅ 发送成功后，从后端重新获取消息列表
    const history = await chatApi.getHistory(conversationId);
    messages.value = history;
  } catch (e) {
    // 失败时显示错误
  }
}
```

**方案 B：使用乐观更新 + 回滚机制**

```typescript
// src/stores/chat.ts - sendMessage 函数
async function sendMessage(content: string) {
  const conversationId = currentConversationId.value!;

  // 乐观更新：立即显示用户消息
  const tempMessage = {
    id: crypto.randomUUID(),
    conversationId,
    role: 'user' as const,
    content,
    createdAt: new Date().toISOString(),
  };
  messages.value.push(tempMessage);

  isLoading.value = true;

  try {
    await chatApi.send(conversationId, content);
    // ✅ 发送成功后更新消息
    const history = await chatApi.getHistory(conversationId);
    messages.value = history;
  } catch (e) {
    // ❌ 失败时回滚：移除临时消息
    const idx = messages.value.findIndex(m => m.id === tempMessage.id);
    if (idx !== -1) {
      messages.value.splice(idx, 1);
    }
    console.error('[ChatStore] 发送失败:', e);
  } finally {
    isLoading.value = false;
  }
}
```

**方案 C：添加消息同步状态标志**

### 问题 5: 监听器初始化时机

#### 问题描述

`listenersInitialized` 是模块级变量，在某些场景下（如页面热重载）可能不会重置，导致监听器重复注册或跳过注册。

#### 当前代码

```typescript
// src/stores/chat.ts
let listenersInitialized = false;  // 模块级变量

function setupListeners() {
  if (listenersInitialized) {
    console.log('[ChatStore] 监听器已初始化，跳过');
    return;
  }
  listenersInitialized = true;
  // ... 注册监听器
}
```

#### 问题分析

| 场景 | listenersInitialized 状态 | 结果 |
|-----|-------------------------|------|
| 首次加载页面 | false | ✅ 正常注册 |
| 切换路由后返回 | true | ✅ 跳过 |
| 页面热重载 (HMR) | true | ❌ 跳过，但监听器可能已失效 |
| 刷新页面 | false | ✅ 正常注册 |

#### 修改方案

**方案 A：使用 ref 替代模块级变量**

```typescript
// src/stores/chat.ts
import { ref } from 'vue';

export const useChatStore = defineStore('chat', () => {
  const listenersInitialized = ref(false);
  // ...
});
```

**方案 B：在 onUnmounted 中重置**

```typescript
// src/views/ChatView.vue
import { onUnmounted } from 'vue';

onMounted(async () => {
  await chatStore.loadConversations();
  chatStore.setupListeners();
});

onUnmounted(() => {
  // ✅ 组件卸载时重置标志
  chatStore.resetListeners();
});
```

**方案 C：每次调用都重新注册，但先清理旧的**

```typescript
// src/stores/chat.ts
function setupListeners() {
  // ✅ 始终清理旧的监听器（虽然 preload 已经做了，但防御性编程）

  });
}
```
