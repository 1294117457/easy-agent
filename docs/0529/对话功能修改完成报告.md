### 问题 3: 消息重复添加到历史 ✅

**修改文件**: `electron/core/application/AgentService.ts`

**问题原因**:

原代码中，`history.slice(0, -1)` 排除了最后一条消息，然后再 `push` 当前用户消息。但当后端已经存储了用户消息时（前端先调用了 `appendMessage`），会导致用户消息被添加两次。

**修改方案**:

```typescript
// ✅ 新增：检查最后一条是否等于当前用户消息
const lastMessage = history[history.length - 1];
const isLastMessageDuplicate =
  lastMessage?.role === 'user' &&
  lastMessage?.content === userInput;

console.log('[AgentService] 是否重复:', isLastMessageDuplicate);

// ✅ 根据是否重复决定使用的历史
const historyToUse = isLastMessageDuplicate
  ? history.slice(0, -1)
  : history;
```

**日志示例**:

```
[AgentService] 历史消息数: 3
[AgentService] 最后一条消息: { role: "user", content: "你好" }
[AgentService] 是否重复: true
[AgentService] 历史消息数量: 2，使用: 2
```

### 问题 4: 前后端状态不同步 ✅

**修改文件**:

| 文件 | 修改内容 |
|-----|---------|
| `electron/ipc/chat.handler.ts` | 添加消息同步事件 |
| `electron/preload.ts` | 添加 `onMessagesSynced` 监听器 |
| `src/types/electron.d.ts` | 添加类型声明 |
| `src/api/chat.ts` | 传递 `onMessagesSynced` 回调 |
| `src/stores/chat.ts` | 接收同步事件，智能合并消息 |

**架构图**:

```
┌─────────────────────────────────────────────────────────────────┐
│                      消息同步流程                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  1. 前端发送消息                                                  │
│     └── 乐观更新：立即显示用户消息                                   │
│     └── 调用 chatApi.send()                                       │
│                                                                  │
│  2. 后端处理                                                      │
│     └── AgentService 存储用户消息                                  │
│     └── 调用 LLM 获取响应                                         │
│     └── AgentService 存储助手消息                                  │
│                                                                  │
│  3. 后端发送同步事件                                               │
│     └── onDone 时发送 agent:messages-synced                        │
│     └── 携带最后两条消息（用户+助手）                                │
│                                                                  │
│  4. 前端接收同步事件                                               │
│     └── onMessagesSynced 回调                                      │
│     └── 增量同步：只添加缺失的消息                                   │
│                                                                  │
│  5. 后台静默同步（额外保险）                                        │
│     └── 100ms 后从后端获取完整历史                                  │
│     └── 智能合并：只添加 ID 不存在的消息                              │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

**性能优化**:

- **乐观更新**：用户消息立即显示，不等待后端
- **流式渲染**：助手消息通过 onToken 实时追加
- **增量同步**：只添加缺失的消息，不重绘整个列表
- **静默同步**：后台执行，不阻塞 UI

### 问题 5: 监听器初始化时机 ✅

**修改文件**: `src/stores/chat.ts`

**修改内容**:

1. 将模块级变量 `let listenersInitialized = false` 改为 Vue ref

```typescript
// ❌ 修改前
let listenersInitialized = false;

// ✅ 修改后
const listenersInitialized = ref(false);
```

2. 添加了 `resetListeners()` 方法

```typescript
function resetListeners() {
  listenersInitialized.value = false;
  console.log('[ChatStore] 监听器标志已重置');
}
```

3. 更新日志输出

```typescript
console.log('[ChatStore] setupListeners 调用, 当前 listenersInitialized:', listenersInitialized.value);
```
