```
我设想实现的功能是一个应用，
	有一个agent核心，然后基于mcp接入各种功能插件，然后可以对功能插件编排实现工作流；
	进一步就是更易操作，比如拖拽编排等等

这里就是基于electron制作应用，LangGraph制作agent核心，
整体我想用六边形架构，同时使用面向对象的思想，你觉得怎么样
```

```
Electron 应用外壳
  └── 主进程
        └── 六边形架构核心
              ├── 内核：AgentCore (LangGraph 编排 + 业务逻辑)
              ├── 端口：抽象接口
              └── 适配器：
                    ├── MCP 适配器 → 启动子进程
                    ├── 持久化适配器 → SQLite
                    ├── LLM 适配器 → API 调用
                    └── IPC 适配器 → 与渲染进程通信
  └── 渲染进程 (Vue UI)
```

| 通信      | 连接              | 方向     | 传输内容                       |
| :-------- | :---------------- | :------- | :----------------------------- |
| **IPC**   | 渲染进程 ↔ 主进程 | 双向     | 用户输入、Agent 响应、配置更新 |
| **stdio** | 主进程 ↔ MCP 插件 | 双向     | 工具调用指令、执行结果         |
| **HTTP**  | 主进程 ↔ LLM API  | 单向请求 | API Key、Prompt、流式响应      |
