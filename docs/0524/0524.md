##### 个人技术

```
主流的前端vue，后端java，LangChain/LangGraph
常见中间件：nginx，mysql，minio，redis
```



##### 功能

```
1.个性化健身助手
	核心功能
		数据采集、个性化计划、教程/动作库
	辅助功能
		打卡+提醒、进度可视化、社交激励
2.结合用户周围健身房数据，给用户生成更全面的健身计划（包括去哪里健身更方便）
	核心功能：
		周围健身房获取、路线规划
3.健身相关物资获取助手，比如教程、蛋白粉、护具
	核心功能：
		物资比价
```

##### 定位

```
1.非盈利
	比如物资比较只提供途径，不参与实际交易
	不介入广告，纯净的服务
2.去中心化
	提供一个agent应用，配置好微型的sqlite数据库
	功能实现基于应用层，比如获取用户硬件权限，自动化帮助用户操作电商app，查看价格并统计
	用户数据都是通过用户设备获取，或者用户同意情况下调用接口获取
3.增加共享数据服务
	但是不是开发中统一提供，而是用户自发组成
	比如一个区域内，有用户想主动组成一个健身共享的集群，他们选择在这共享健身房、物资等数据，进行交流
	同时也维护非盈利的概念，避免类似许多主流app一样，发展到最后，为了盈利功能差广告多
	
4.共享数据的安全考虑
	用户只是自己使用时，可以说只获取外界数据，可以随意操作自己的情况
	如果用户在共享区域活动，对应行为不受任何中心化平台监督管理，而是其操作都类似web3的概念一样，有永久性的共识
	也维护去中心化的理念，任何人都可以独立于共享服务单独使用，但是如果要使用共享服务，要有区块链一样的共识机制，避免用户随意污染公共数据
5.开源项目
```
