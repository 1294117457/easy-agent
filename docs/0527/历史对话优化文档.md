# 历史对话优化方案文档

## 一、需求概述

### 1.1 当前问题
- 对话标题显示为"新对话"，没有根据内容生成
- 历史对话列表缺少开始和结束时间
- 长对话没有压缩机制，会导致性能问题和token消耗增加

### 1.2 优化目标
1. **自动压缩**：所有对话每20轮进行一次压缩
2. **关闭时处理**：对话每次关闭时，进行压缩并生成标题
3. **历史展示**：历史对话只显示标题、开始时间、结束时间，不需要内容概括

### 3.3 压缩服务 (`electron/core/application/CompressionService.ts`)

新建压缩服务，负责对话压缩和标题生成：

```typescript
import type { IStoragePort } from '../ports/storage.port.js';
import type { I_llmPort } from '../ports/llm.port.js';

export interface CompressionResult {
  summary: string;
  title: string;
}

export class CompressionService {
  constructor(
    private storage: IStoragePort,
    private llm: I_llmPort
  ) {}

  /**
   * 检查是否需要压缩（每20轮压缩一次）
   */
  shouldCompress(conversationId: string): boolean {
    const count = this.storage.getMessageCount(conversationId);
    return count > 0 && count % 20 === 0;
  }

  /**
   * 压缩对话
   * 1. 获取所有消息
   * 2. 调用 LLM 生成摘要
   * 3. 保留最后5轮消息，其余标记为已压缩
   */
  async compressConversation(conversationId: string): Promise<CompressionResult | null> {
    const messages = this.storage.getMessages(conversationId);
    if (messages.length < 10) return null;

    // 提取最近的消息（保留最后5轮 = 10条消息）
    const recentMessages = messages.slice(-10);
    const historicMessages = messages.slice(0, -10);

    // 构造压缩提示
    const summaryPrompt = this.buildSummaryPrompt(historicMessages);
    const summaryResult = await this.llm.complete(summaryPrompt);

    // 解析结果
    const result = this.parseSummaryResult(summaryResult);

    // 更新数据库
    this.storage.updateConversationSummary(conversationId, result.summary);

    // 标记历史消息为已压缩
    for (const msg of historicMessages) {
      this.storage.updateMessageCompressed(msg.id, true);
    }

    return result;
  }

  /**
   * 生成对话标题
   */
  async generateTitle(conversationId: string): Promise<string> {
    const messages = this.storage.getMessages(conversationId);
    const firstUserMessage = messages.find(m => m.role === 'user');

    if (firstUserMessage) {
      // 如果有用户消息，用前20字符作为标题
      const title = firstUserMessage.content.substring(0, 20);
      return title + (firstUserMessage.content.length > 20 ? '...' : '');
    }

    // 否则使用 LLM 生成
    const titlePrompt = this.buildTitlePrompt(messages);
    const titleResult = await this.llm.complete(titlePrompt);
    return this.parseTitleResult(titleResult);
  }

  /**
   * 结束对话（压缩 + 生成标题）
   */
  async endConversation(conversationId: string): Promise<void> {
    // 1. 检查并压缩
    if (this.shouldCompress(conversationId)) {
      await this.compressConversation(conversationId);
    } else {
      // 即使不压缩，也生成摘要（保留所有消息引用）
      const messages = this.storage.getMessages(conversationId);
      const summary = this.buildAutoSummary(messages);
      this.storage.updateConversationSummary(conversationId, summary);
    }

    // 2. 生成标题
    const title = await this.generateTitle(conversationId);

    // 3. 标记对话结束
    this.storage.endConversation(conversationId, title);
  }

  /**
   * 构建摘要生成提示
   */
  private buildSummaryPrompt(messages: Message[]): string {
    const content = messages
      .map(m => `${m.role}: ${m.content}`)
      .join('\n');

    return `请为以下对话生成一个简洁的摘要（不超过200字）和一个简短标题（不超过10字）。

格式要求：
摘要：<200字的对话摘要>
标题：<10字以内的标题>

对话内容：
${content}

请按以下格式返回：
摘要：<你的摘要>
标题：<你的标题>`;
  }

  /**
   * 解析摘要结果
   */
  private parseSummaryResult(result: string): CompressionResult {
    const summaryMatch = result.match(/摘要[：:]\s*(.+?)(?=标题|$)/s);
    const titleMatch = result.match(/标题[：:]\s*(.+?)(?=$)/s);

    return {
      summary: summaryMatch?.[1]?.trim() || '对话摘要',
      title: titleMatch?.[1]?.trim() || '未命名对话'
    };
  }

  /**
   * 构建标题生成提示
   */
  private buildTitlePrompt(messages: Message[]): string {
    const preview = messages
      .slice(0, 6)  // 取前6条消息
      .map(m => `${m.role}: ${m.content.substring(0, 100)}`)
      .join('\n');

    return `根据以下对话内容，生成一个简短的标题（不超过10字）：

${preview}

直接返回标题，不需要其他内容。`;
  }

  /**
   * 解析标题结果
   */
  private parseTitleResult(result: string): string {
    const title = result.trim();
    return title.length > 10 ? title.substring(0, 10) + '...' : title;
  }

  /**
   * 自动生成摘要（不调用 LLM）
   */
  private buildAutoSummary(messages: Message[]): string {
    const userMessages = messages.filter(m => m.role === 'user');
    const assistantMessages = messages.filter(m => m.role === 'assistant');

    return `共 ${userMessages.length} 轮对话，涉及 ${assistantMessages.length} 次回复。`;
  }
}
```

/**
 * 格式化时间范围显示
 * 格式：开始时间 - 结束时间 或 开始时间 - 进行中
 */
function formatTimeRange(startAt: string, endedAt?: string): string {
  const start = formatDate(startAt);

  if (endedAt) {
    const end = formatTime(endedAt);
    return `${start} - ${end}`;
  }

  return `${start} - 进行中`;
}

/**
 * 格式化日期
 */
function formatDate(dateStr: string) {
  const date = new Date(dateStr);
  const now = new Date();
  const diff = now.getTime() - date.getTime();
  const oneDay = 24 * 60 * 60 * 1000;

  if (diff < oneDay && date.getDate() === now.getDate()) {
    return date.toLocaleTimeString('zh-CN', { hour: '2-digit', minute: '2-digit' });
  } else if (diff < 2 * oneDay) {
    return '昨天 ' + date.toLocaleTimeString('zh-CN', { hour: '2-digit', minute: '2-digit' });
  } else if (diff < 7 * oneDay) {
    return date.toLocaleDateString('zh-CN', { weekday: 'short', hour: '2-digit', minute: '2-digit' });
  } else {
    return date.toLocaleDateString('zh-CN', { month: 'short', day: 'numeric', hour: '2-digit', minute: '2-digit' });
  }
}

## 五、触发时机

### 5.1 自动压缩（每20轮）
- **触发点**：用户发送消息后
- **判断条件**：`messageCount % 20 === 0`
- **执行内容**：
  1. 获取最近 20 轮消息之前的消息
  2. 调用 LLM 生成摘要
  3. 标记旧消息为已压缩
  4. 更新对话摘要

### 5.2 关闭对话时
- **触发点**：用户点击"新建对话"或切换到其他对话时
- **判断条件**：当前对话有消息且未结束
- **执行内容**：
  1. 检查是否需要压缩（执行压缩逻辑）
  2. 生成对话标题
  3. 记录结束时间
  4. 刷新对话列表

### 5.3 标题生成逻辑
1. **优先使用第一条用户消息**：取前20字符
2. **备选方案**：调用 LLM 根据对话内容生成（不超过10字）

## 九、注意事项

### 9.1 性能考虑
- 压缩操作是异步的，不阻塞用户操作
- LLM 调用有超时保护
- 压缩失败时记录日志但不中断流程

### 9.2 数据安全
- 所有数据存储在本地 SQLite
- API Key 加密存储
- 对话内容不上传

### 9.3 用户体验
- 压缩过程对用户透明
- 历史对话只读，不可编辑
- 结束对话后显示提示
