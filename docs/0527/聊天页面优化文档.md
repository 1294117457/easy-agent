## 一、问题清单

| # | 问题 | 描述 | 优先级 |
|---|------|------|--------|
| 1 | 对话超出部分不显示 | 消息列表无法滚动，内容被遮挡 | 🔴 高 |
| 2 | 下拉框无法滚动 | 点击选择器后，下拉框内容无法滚动 | 🟡 中 |
| 3 | 模型和 Prompt 合并在一起 | 两个选择器共用一个下拉框 | 🟡 中 |
| 4 | 选择器位置不对 | 应放在 MCP 按钮旁边 | 🟡 中 |

## 二、问题分析

### 2.1 对话超出部分不显示

**问题原因**：

```css
/* 当前 MessageList.vue */
.message-list {
  flex: 1;
  overflow-y: auto;
  /* ... */
}

/* 当前 ChatView.vue */
.messages-container {
  flex: 1;
  overflow: hidden;  /* ← 问题：隐藏溢出 */
  padding: 0 16px;
}
```

**解决方案**：
1. `messages-container` 需要能够容纳滚动内容
2. 设置 `overflow: hidden` 会阻止内部滚动
3. 改为让 `MessageList` 自己滚动

### 2.2 下拉框无法滚动

**问题原因**：
```css
.dropdown-panel {
  max-height: 480px;
  overflow: hidden;  /* ← 问题：没有设置 overflow */
}
```

**解决方案**：
```css
.dropdown-panel {
  max-height: 480px;
  overflow-y: auto;  /* ← 允许垂直滚动 */
}
```

### 2.3 模型和 Prompt 合并在一起

**当前问题**：两个选择器共用一个下拉框

```vue
<!-- 当前代码：两个点击都触发同一个事件 -->
<div class="selector-item" @click="toggleModelDropdown">  <!-- 模型 -->
<div class="selector-item" @click="toggleModelDropdown">  <!-- Prompt -->
```

**解决方案**：
1. 拆分为两个独立的切换函数
2. 拆分为两个独立的下拉框组件或一个组件两个状态

### 2.4 选择器位置不对

**当前布局**：
```
<ChatHeader>
  <toggle-btn> <title> <actions>  <!-- 选择器在这里 -->
</ChatHeader>
<ModelPromptSelector />  <!-- 选择器在 Header 下面 -->
```

**期望布局**：
```
<ChatHeader>
  <toggle-btn> <title> <ModelSelector> <PromptSelector> <MCPBtn>
</ChatHeader>
```

## 三、优化方案

### 3.1 界面布局优化

**期望布局**：
```
┌─────────────────────────────────────────────────────────┐
│ ☰  对话标题   [模型▼] [Prompt▼]                    MCP │  ← Header 内
├─────────────────────────────────────────────────────────┤
│                                                         │
│                   消息列表区域                            │  ← 可滚动
│                   （可滚动）                             │
│                                                         │
├─────────────────────────────────────────────────────────┤
│ [输入框]                                         [发送] │
└─────────────────────────────────────────────────────────┘
```

### 3.2 组件拆分

**方案 A：将选择器集成到 ChatHeader 中**

修改 `ChatHeader.vue`：
```vue
<template>
  <header class="chat-header">
    <button class="toggle-btn">☰</button>
    <span class="title">对话标题</span>

    <!-- 模型选择器 -->
    <ModelSelector />

    <!-- Prompt 选择器 -->
    <PromptSelector />

    <button class="mcp-btn">MCP</button>
  </header>
</template>
```

**方案 B：拆分为两个独立组件**

- `ModelSelector.vue`：模型选择器
- `PromptSelector.vue`：Prompt 选择器

**推荐方案 A**，保持简洁。

### 3.3 下拉框滚动修复

```css
.dropdown-panel {
  max-height: 320px;
  overflow-y: auto;
  scrollbar-width: thin;
  scrollbar-color: #d1d5db transparent;
}
```

### 3.4 消息列表滚动修复

**当前结构**：
```
.chat-main
├── ChatHeader (固定高度)
├── ModelPromptSelector (固定高度)
├── .messages-container (flex: 1, overflow: hidden)
│   └── .message-list (overflow-y: auto)
└── .input-container (固定高度)
```

**问题**：`messages-container` 设置了 `overflow: hidden`，阻止了 `message-list` 的滚动。

**解决方案**：
```css
.messages-container {
  flex: 1;
  min-height: 0;  /* 关键：允许 flex 收缩 */
  display: flex;
  flex-direction: column;
  overflow: hidden;
}

.message-list {
  flex: 1;
  overflow-y: auto;
  min-height: 0;  /* 关键：允许在 flex 中滚动 */
}
```

import { useConfigStore } from '@/stores/config';
import { useRouter } from 'vue-router';

const configStore = useConfigStore();
const router = useRouter();

const showDropdown = ref(false);

function toggleDropdown(event: MouseEvent) {
  const rect = (event.currentTarget as HTMLElement).getBoundingClientRect();
  // 可以在这里设置下拉框位置
  showDropdown.value = !showDropdown.value;
}

function closeDropdown() {
  showDropdown.value = false;
}

}

## 七、注意事项

1. **滚动穿透**：下拉框打开时，点击外部应关闭
2. **位置计算**：下拉框应紧贴在按钮下方
3. **响应式**：确保在不同屏幕宽度下都能正常显示
4. **性能**：使用 `Teleport` 避免定位问题
