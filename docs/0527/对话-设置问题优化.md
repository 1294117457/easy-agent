## 二、问题清单

### 问题 1：历史对话数据未正确继承

#### 2.1 问题描述

切换历史对话后，新发送的消息实际使用的是旧对话的数据，导致对话内容混乱。

#### 2.2 复现步骤

1. 在「对话 A」中发送消息
2. 切换到「对话 B」
3. 在「对话 B」中发送消息
4. 观察：消息可能被错误地关联到「对话 A」

#### 2.3 根因分析

需要检查以下代码流程：

```typescript
// src/stores/chat.ts
async function sendMessage(content: string) {
  // 问题：这里直接使用 currentConversationId.value
  // 如果切换对话后异步操作未完成，可能获取到旧值
  await chatApi.send(currentConversationId.value!, content);
}
```

可能的问题点：
1. `currentConversationId` 更新是异步的，但 `sendMessage` 可能在更新完成前执行
2. 消息列表 `messages` 没有在切换对话时正确清空/重置
3. `selectConversation` 方法中没有等待消息加载完成

#### 2.4 修复方案

**方案 A：确保状态同步**

```typescript
async function selectConversation(id: string) {
  // 1. 先清空当前消息，防止闪烁
  messages.value = [];
  currentConversationId.value = id;

  // 2. 等待消息加载完成后再返回
  const history = await chatApi.getHistory(id);
  messages.value = history;
}

async function sendMessage(content: string) {
  // 验证 conversationId
  const conversationId = currentConversationId.value;
  if (!conversationId) {
    console.error('[ChatStore] 没有选中的对话');
    return;
  }

  // 添加用户消息
  const userMsg: Message = {
    id: crypto.randomUUID(),
    conversationId,
    role: 'user',
    content,
    createdAt: new Date().toISOString(),
  };
  messages.value.push(userMsg);

  isLoading.value = true;

  try {
    await chatApi.send(conversationId, content);
  } catch (e) {
    isLoading.value = false;
    // 可以在这里移除刚添加的用户消息
    messages.value.pop();
  }
}
```

**方案 B：添加 loading 状态防止竞态**

```typescript
const isSwitchingConversation = ref(false);

async function selectConversation(id: string) {
  isSwitchingConversation.value = true;
  currentConversationId.value = id;
  messages.value = [];
  messages.value = await chatApi.getHistory(id);
  isSwitchingConversation.value = false;
}

async function sendMessage(content: string) {
  if (isSwitchingConversation.value) {
    console.warn('[ChatStore] 正在切换对话，忽略发送');
    return;
  }
  // ... 发送逻辑
}
```

#### 2.5 验证方法

1. 创建多个对话，分别发送不同内容
2. 切换对话，确认消息列表正确显示
3. 在每个对话中发送新消息，确认消息正确归属

### 问题 2：API Key 列表高度无限增长

#### 2.1 问题描述

每次添加新的 API Key 时，整个列表区域的高度不断增加，没有限制。

#### 2.2 当前代码

```vue
<!-- src/views/SettingsView.vue -->
<div class="api-keys-list">
  <div v-for="key in configStore.apiKeys" :key="key.id" class="api-key-item">
    ...
  </div>
</div>
```

#### 2.3 修复方案

给 `api-keys-list` 添加固定高度和滚动：

```vue
<div class="api-keys-list">
  <div
    v-for="key in configStore.apiKeys"
    :key="key.id"
    class="api-key-item"
  >
    ...
  </div>
  <div v-if="configStore.apiKeys.length === 0" class="empty-hint">
    暂无配置的 API Key，请添加
  </div>
</div>
```

```css
.api-keys-list {
  max-height: 240px;  /* 固定最大高度 */
  overflow-y: auto;   /* 超出滚动 */
  margin-bottom: 16px;
  /* 美化滚动条 */
  scrollbar-width: thin;
  scrollbar-color: #ccc transparent;
}

.api-keys-list::-webkit-scrollbar {
  width: 6px;
}

.api-keys-list::-webkit-scrollbar-thumb {
  background: #ccc;
  border-radius: 3px;
}

.api-keys-list::-webkit-scrollbar-track {
  background: transparent;
}
```

#### 2.4 预期效果

- 列表最大高度 240px
- 超出后显示细滚动条
- 滚动条样式与整体设计协调

### 问题 3：Prompt 模板功能不可用

#### 3.1 问题描述

Prompt 配置界面存在，但可能未正确连接到后端。

#### 3.2 需要检查的代码

**前端 Store（config.ts）**

```typescript
// src/stores/config.ts
async function createPrompt(data: {
  name: string;
  description?: string;
  systemPrompt: string;
}) {
  const created = await configApi.createPrompt(data);
  prompts.value.push(created);
  return created;
}

async function deletePrompt(id: string) {
  await configApi.deletePrompt(id);
  prompts.value = prompts.value.filter((p) => p.id !== id);
}
```

**API 调用（config.ts API）**

```typescript
// src/api/config.ts
export const configApi = {
  createPrompt: (data: { name: string; description?: string; systemPrompt: string }) =>
    window.electronAPI.createPrompt(data),
  deletePrompt: (id: string) => window.electronAPI.deletePrompt(id),
  // ...
};
```

**IPC Handler（config.handler.ts）**

```typescript
// electron/ipc/config.handler.ts
ipcMain.handle('config:prompt:create', (_, data) => {
  return storage.createPrompt(data);
});

ipcMain.handle('config:prompt:delete', (_, id) => {
  return storage.deletePrompt(id);
});
```

**Storage 实现**

需要检查 `storage.createPrompt` 和 `storage.deletePrompt` 是否正确实现。

#### 3.3 测试方法

1. 在设置页面添加一个 Prompt
2. 检查浏览器控制台是否有错误
3. 刷新页面，检查 Prompt 是否持久化
4. 删除 Prompt，验证删除功能

#### 3.4 可能的问题

1. **API 未注册**：`window.electronAPI.createPrompt` 未定义
2. **Storage 方法缺失**：`storage.createPrompt` 未实现
3. **数据结构不匹配**：前后端数据结构不一致

### 问题 4：历史对话滚动条样式不协调

#### 4.1 问题描述

当前滚动条默认样式较宽，与整体 UI 不协调。

#### 4.2 当前代码

```css
/* ConversationList.vue */
.list {
  flex: 1;
  overflow-y: auto;
}
```

#### 4.3 修复方案

```css
.list {
  flex: 1;
  overflow-y: auto;
  /* 滚动条样式 */
  scrollbar-width: thin;           /* Firefox */
  scrollbar-color: #d1d5db transparent;  /* Firefox */
}

/* Webkit 浏览器滚动条样式 */
.list::-webkit-scrollbar {
  width: 6px;
}

.list::-webkit-scrollbar-track {
  background: transparent;
}

.list::-webkit-scrollbar-thumb {
  background: #d1d5db;
  border-radius: 3px;
}

.list::-webkit-scrollbar-thumb:hover {
  background: #9ca3af;
}
```

#### 4.4 预期效果

- 滚动条宽度从默认 12-16px 减少到 6px
- 灰色调，与浅色背景协调
- hover 时颜色加深
